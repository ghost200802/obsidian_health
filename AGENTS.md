LLM Wiki - Agent 工作规则

## 仓库概述

这是一个 LLM Wiki 知识库，不是普通的笔记仓库。
资料进来后，需要经过 ingest → 知识编译 → 沉淀为 wiki 页面的完整流程。
目标是让知识越用越好用，而不是只做一次性的摘要。

## 目录结构

```
llm-wiki/
├── raw/          # 原始资料层（只读输入）
├── wiki/         # 知识沉淀层（Agent 的主要产出区）
└── AGENTS.md     # 本文件 - Agent 工作规则
```

## 分层规则

### raw/ - 原始资料层
- 存放原始文章、网页摘录、PDF 笔记、会议纪要等
- 命名规范：`YYYY-MM-DD-主题关键词.md`
- Agent 不应修改 raw/ 中的文件内容
- raw/ 中的文件是输入，不是产出

### wiki/ - 知识沉淀层
- 存放经过整理和编译的知识页面
- 页面类型包括：
  - **来源页** `source-*.md`：对 raw 资料的结构化整理
  - **概念页**：从资料中提取的核心概念
  - **主题页**：围绕某一主题的聚合知识
  - **工具页**：工具、框架、技术栈的整理
  - **对比页**：概念之间的对比分析
  - **索引页**：某一领域的入口和导航

## Ingest 规则

当用户要求 ingest 一篇 raw 资料时，按以下步骤执行：

### 1. 阅读并理解
- 仔细阅读 raw/ 中的源文件
- 提取核心观点、关键概念、重要论据

### 2. 生成来源页（wiki/source-*.md）
- 在 wiki/ 下创建来源页
- 包含以下内容：
  - 文章标题和元信息（作者、来源、日期）
  - 原文链接
  - 核心摘要（3-5 句话）
  - 关键观点列表
  - 值得保留的原因
  - 与已有知识页的关联

### 3. 提取概念，创建或更新概念页
- 从资料中识别出值得独立记录的概念
- 如果 wiki/ 中已有相关概念页，则更新它（追加新信息，标注来源）
- 如果是新概念，创建新的概念页，包含：
  - 概念定义
  - 核心要点
  - 与其他概念的关系
  - 相关来源链接

### 4. 添加交叉链接
- 使用 Obsidian 的 `[[]]` 双链语法
- 每个生成的页面至少应该链接到 2-3 个其他页面
- 链接应该是有意义的关联，不是随意堆砌

### 5. 更新索引页
- 如果某个领域的页面数量增多，创建或更新该领域的索引页
- 索引页提供该领域所有相关页面的导航入口

## 查询规则

当用户提问时：
1. **优先查阅 wiki/ 层**：先看已有的知识页面，站在已整理的知识上回答
2. **补充查阅 raw/ 层**：如果 wiki 层信息不足，再去 raw/ 中查找原始资料
3. **回答时标注来源**：引用 wiki 页面时使用 `[[]]` 链接

## 回写规则

当一次问答产生了高价值的知识时：
- 将问答中产生的对比、总结、新理解沉淀回 wiki/
- 更新相关概念页，补充新视角
- 确保新页面与已有页面建立链接关系

## 写作规范

- 使用中文撰写（除非原始资料为英文且术语需保留原文）
- 专业术语首次出现时标注英文原文
- 每个页面顶部使用 frontmatter 标注元信息：
  ```yaml
  ---
  type: concept | source | topic | tool | comparison | index
  created: YYYY-MM-DD
  updated: YYYY-MM-DD
  tags: [tag1, tag2]
  ---
  ```
- 页面内容使用清晰的标题层级
- 使用列表、表格等结构化格式提升可读性

## 禁止事项

- 不要修改 raw/ 中的文件
- 不要删除 wiki/ 中的已有页面（只做更新和追加）
- 不要创建没有链接关系的孤立页面
- 不要在 ingest 时跳过概念提取步骤，只做摘要
