# Wiki 目录结构重组计划

## 摘要

将 `wiki/` 目录从最大 4 层嵌套重组为最大 2 层，消除纯容器目录和散落文件，合并小类别，使结构更扁平、更清晰。所有文件操作使用 `git mv` 移动，不重新写入文件。

## 当前状态分析

| 问题 | 位置 | 说明 |
|------|------|------|
| 深度超标 | `应用场景/减重管理/减重方法/`、`应用场景/减重管理/特殊人群减重/` | 4层嵌套，超出2层限制 |
| 纯容器目录 | `应用场景/减重管理/` | 0个直接文件，仅作为中间层 |
| 散落文件 | `营养学基础/`(2)、`营养素/`(4)、`食物分类/`(12) | 文件未归入子目录 |
| 归类异常 | `营养素/植物化学物/宏量营养素.md`、`微量营养素.md` | 索引页不属于植物化学物 |
| 小类别碎片化 | 疾病营养下7个子目录，其中3个<10文件 | 需重新分类 |

| 新目录 | 内容来源 | 文件数 | 分类逻辑 |
|--------|----------|--------|----------|
| `疾病营养/器官系统/` | 器官系统(37) + 精神心理疾病(9) | **46** | 按器官/系统分类的具体疾病。精神心理疾病主要是神经系统疾病（帕金森、阿尔茨海默、癫痫、脑卒中），与器官系统中已有的"神经系统疾病营养"同属一类 |

## 假设与决策

1. **Obsidian [[]] 链接安全**：基于文件名而非路径，移动不会破坏交叉引用，无需修改文件内容
2. **frontmatter tags 不修改**：标签独立于目录结构
3. **使用 `git mv` 移动文件**：不重新写入，保留 git 历史
4. **索引页后续更新**：移动完成后需检查 index 类型页面的导航内容

## 验证步骤

1. 移动完成后检查所有目录深度 ≤ 2（wiki/ 为 level 0）
2. 确认无纯容器目录（每个目录都有直接文件）
3. 确认无散落文件（所有文件都在子目录中）
4. 在 Obsidian 中打开验证链接完整性
5. 检查 `git status` 确认所有移动正确记录
