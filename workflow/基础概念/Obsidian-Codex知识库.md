## 核心摘要

本文提出了一种基于 [[Obsidian知识库|Obsidian]] + [[Codex智能体]] 的 [[LLM-Wiki知识管理]] 知识管理方案。核心理念是将知识库从"资料仓库"升级为"知识编译系统"，通过 Agent 驱动的 ingest 流程，让原始资料经过整理、提取、关联后沉淀为可复用的中间知识层。

## 关键观点

1. **知识库不等于仓库**：资料越存越多但不好用，问题在于缺乏持续的知识消化机制
2. **编译知识而非存储资料**：通过 Agent 自动整理，将原始资料转化为结构化的知识页面
3. **分层是关键**：[[Raw层]] 存原始资料，[[Wiki层]] 存编译后的知识，两层必须分离
4. **链路比规模重要**：第一次应验证 raw → ingest → wiki → query → 回写 的完整链路
5. **与 RAG 的本质区别**：[[LLM-Wiki知识管理]] 强调知识持续累积和复利，[[检索增强生成]] 强调按需检索

## 为什么值得保留

- 提供了最小可运行的 [[LLM-Wiki知识管理]] 方案，适合快速上手
- 强调"先跑通链路，再优化结构"的务实思路
- 清晰的 [[Ingest工作流]] 设计原则
