# LLM Wiki

## 定义

LLM Wiki 是一种由 LLM Agent 驱动的知识管理方法论。核心理念是将知识库从被动的"资料仓库"升级为主动的"知识编译系统"，通过 Agent 的持续整理，让原始资料转化为可复用、可累积、可生长的结构化知识。

## 核心特征

1. **分层架构**：[[Raw层]]（原始资料）与 [[Wiki层]]（编译后的知识）严格分离
2. **Agent 驱动**：通过 Agent（如 [[Codex智能体]]）自动执行 [[Ingest工作流]]
3. **知识编译**：不只是摘要，而是提取概念、建立关联、更新知识网络
4. **持续进化**：新资料和新问答都会沉淀回知识库，形成复利效应
5. **优先查 wiki**：提问时优先站在已整理的知识层回答，而非每次从原始文档拼凑

## 工作流

```
资料收集 → raw/ 存入 → Agent ingest → wiki/ 沉淀 → 基于 wiki 提问 → 高价值问答回写 wiki
```

## 与其他方案对比

| 维度 | [[LLM-Wiki知识管理]] | [[检索增强生成]] | 传统笔记 |
|------|-------------|---------|---------|
| 知识处理 | 主动编译 | 按需检索 | 手动整理 |
| 知识累积 | 持续复利 | 不累积 | 依赖人 |
| 回答质量 | 站在知识层回答 | 从原始文档拼凑 | 依赖记忆 |
| 维护成本 | Agent 自动 | 向量库维护 | 全手动 |
