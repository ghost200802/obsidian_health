# 客户交付文档生成工作流

## 目标

将体检报告解读和 `1v1 定制方案` 直接编译成客户可发送的 Word 文档，同时保留 HTML 预览。客户不需要复制聊天内容，也不需要自己排版。

## 生成方式

内部结构化内容准备好后，运行：

```bash
.venv-docs/bin/python scripts/generate_client_document.py input.json \\
  --out-dir "客户减肥指导/客户名/交付文件"
```

结构化输入示例见 `scripts/client_document.example.json`。实际客户文件由 Agent 根据报告和方案自动组装，不要求客户手动填写这个 JSON。

加上 `--pdf` 会同时生成 PDF。`.docx` 适合后续编辑，`.pdf` 适合直接发送，HTML 用于预览和排版核对。

## 内容边界

- 体检部分遵循 [[健康体检检验报告解读]] 和 `checkup-report` 规则，只写报告支持的结论。
- 1v1 部分遵循 [[1v1方案文风规则]]，先讲客户能感受到的问题，再给可执行动作。
- 文档末尾保留健康管理免责声明，不替代诊疗、处方或药物调整。

## 客户版结构

1. 封面
2. 先看结论
3. 体检报告解读
4. 1v1 定制方案
5. 日常执行清单、复查与说明

## 依赖

本地使用 `.venv-docs`，安装：

```bash
python3 -m venv .venv-docs
.venv-docs/bin/python -m pip install -r scripts/requirements-docs.txt
```

PDF 依赖 macOS 的图形库。若运行 `--pdf` 时提示缺少 `libgobject`、`pango` 或 `cairo`，再运行：

```bash
brew install pango
```
