# Ingest 工作流

## 定义

Ingest 工作流是 [[LLM-Wiki知识管理]] 中 Agent 处理原始资料的标准流程。它不只是生成摘要，而是一个包含阅读理解、来源页生成、概念提取、交叉链接和索引更新的完整知识编译过程。

## 流程步骤

### 1. 阅读并理解
- 仔细阅读 [[Raw层]] 中的源文件
- 提取核心观点、关键概念、重要论据
- 识别与已有知识页的潜在关联

### 2. 生成来源页
- 在 /source 下创建 `source-*.md` 文件
- 包含：元信息、核心摘要、关键观点列表、保留价值、关联知识页

### 3. 提取概念
- 从资料中识别值得独立记录的概念
- 已有概念页 → 追加更新，标注来源
- 新概念 → 创建新页面，包含定义、要点、关联，要求符合[[知识点创建工作流]]

### 4. 添加交叉链接
- 使用 Obsidian `[[]]` 双链语法
- 每个页面至少链接 2-3 个其他页面
- 链接应有意义，不是随意堆砌

### 5. 更新索引页
- 某领域页面增多时，创建或更新索引页
- 索引页提供该领域的导航入口

### 6. 验证并提交
- 检查本次新增或修改的 `source/`、`wiki/` 文件，确认 frontmatter、双链、结构和内容无明显问题
- 如发现问题，先修复，再重新检查
- 确认无问题后，执行一次 git 提交和 push

## 关键原则

- **先跑通链路，再优化规模**：第一次只 ingest 一篇资料
- **任务要具体**：不要说"帮我优化知识库"，要说"请 ingest raw/ 中的 xxx 文件"
- **检查产出**：不只看 Agent 回复的文字，要回 [[Obsidian知识库|Obsidian]] 检查实际文件变动
- **收尾闭环**：完成后不仅要检查文件，还要在确认无问题后提交并 push

## 常见错误

- 只做摘要，不提取概念和建立链接 → 变成自动摘要器
- 跳过交叉链接步骤 → 生成孤立页面
- 一次 ingest 太多资料 → 难以验证链路是否通畅
