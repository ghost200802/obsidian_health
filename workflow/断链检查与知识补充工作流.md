# 断链检查与知识补充工作流

## 定义

从 wiki 页面的 `[[]]` 引用关系出发，扫描断链并分类处理：同物异名则改链，知识缺口则补充，无关引用则清理，新节点递归修复直到无断链。

## 断链分类

| 类别 | 标记 | 判定 | 处理 |
|------|------|------|------|
| **同物异名** | 🔵 | wiki 中已有相似节点，仅命名不同 | 修改引用指向已有节点 |
| **知识缺口** | 🟠 | 属于营养/健康知识体系，值得创建 | 创建新节点（权威、全面、简洁） |
| **无关引用** | ⚪ | 药物名、器官名、通用术语等 | 移除 `[[]]`，保留文本 |
| **畸形链接** | ❌ | 格式错误（如带 `wiki/` 前缀） | 修正格式 |

> Obsidian 短链接（如 `[[蛋白质]]` 指向子目录中的文件）不算断链。

## 处理规则

### A. 同物异名（🔵）— 引用优先于创建

搜索已有相似节点，全局替换所有引用文件中的 `[[旧名]]` → `[[已有页面名]]`。完成后全局验证无残留。

### B. 知识缺口（🟠）— 严格判定是否创建

**创建**：营养素、营养学概念、疾病营养、膳食模式、营养评价工具等知识体系内概念。
**不创建**：通用词汇（代谢、饮食）、药物名（二甲双胍）、器官名（肝脏、肺）、非健康技术（远程医疗）。

创建流程：确定目录位置 → 从 source 层提取内容 → 按 [[知识点创建工作流]] 创建 → 添加 3-5 个出链。

### C. 无关引用（⚪）

`[[无关术语]]` → `无关术语`，保留文本，移除链接格式。

## 递归修复

新节点可能引入新断链，需递归检查其出链：

- 新产生的 🔵/⚪ 断链 → 直接处理
- 新产生的 🟠 断链 → 创建新节点后继续递归
- **递归深度 ≤ 3 层**，防止无限扩展

## 执行策略

**自动连续执行**，按目录优先级逐批处理，直到所有断链清零：

```
全量扫描 wiki/ 断链 → 按目录分组 → 按 P0→P3 优先级逐目录处理
  → 每个目录：扫描 → 分类 → 处理 → 递归修复 → 输出报告
  → 自动进入下一目录，直到所有目录断链清零
```

| 指标 | 上限 |
|------|------|
| 每批断链数 | ≤ 30 |
| 每批新节点数 | ≤ 10 |

目录优先级：`营养素/` → `营养学基础/` → `疾病营养/` → `应用场景/` → `人群营养/` → `膳食/` → `食物分类/`

> 单次任务下达 `"执行断链检查与知识补充工作流"` 即自动处理全部目录。也可指定单个目录单独执行。

## 常见错误

- 一次性全量修复 → 难以验证，应分批
- 同物异名时创建新页面 → 应改链到已有节点
- 过度创建（药物名、器官名也建页面）→ 严格判定，无关的直接清理
- 递归无限扩展 → 限制深度 ≤ 3
- 清理时误删文本 → 只移除 `[[]]`，保留文字内容
