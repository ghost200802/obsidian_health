# Wiki 整理工作流

## 定义

Wiki 整理工作流是 [[LLM-Wiki知识管理]] 中 Agent 对 [[Wiki层]] 进行系统性维护的标准流程。与 [[Ingest工作流]]（从 raw 到 wiki 的正向编译）不同，本工作流聚焦于对已有 wiki 页面的**盘点、充实、合并拆分和关联补全**，是知识库从"有内容"到"高质量知识网络"的关键跃迁。

## 适用场景

- 知识库已积累大量 wiki 页面，需要系统性整理
- 发现部分节点内容不足、重复或孤立
- 需要提升知识网络的密度和一致性
- 新增了大量 source 页面后，需要回填 wiki 节点

## 流程步骤

### 阶段一：盘点与诊断

#### 1.1 全量扫描 wiki 节点

- 遍历 `wiki/` 下所有 `.md` 文件
- 按内容充实度分为三级：

| 级别 | 判定标准 | 标记 |
|------|---------|------|
| **空节点** | 仅有 frontmatter 和标题，无实质内容（<30字正文） | 🔴 |
| **骨架节点** | 有标题结构和少量要点，但缺乏段落展开和数据支撑（<200字正文） | 🟡 |
| **有内容节点** | 有完整段落、数据表格或专业论述 | 🟢 |

- 输出**诊断清单**：按目录分组，列出每个节点的级别、字数、frontmatter 信息

#### 1.2 识别重复与相似节点

- 在全量扫描中同步执行相似度检测：
  - **完全重复**：两个节点文件名不同，但内容实质一致 → 记入"合并候选"
  - **高度相似**：两个节点讨论同一概念，内容有大量重叠 → 记入"合并候选"
  - **主题交叉**：两个节点分别涉及同一主题的不同侧面 → 记入"拆分重组候选"
  - **包含关系**：A 节点的内容完全涵盖 B 节点 → 记入"合并候选"
- **同物异名识别**：同一物质有多个名称（如"烟酸"=维生素B3），需统一命名
  - 检查文件名是否只使用了其中一个名称，而内容中提到了别名（如 `烟酸.md` 首行写"维生素B3"）
  - 检查 wiki 中是否有引用了别名但不存在的文件（如 `[[维生素B6]]` 有 10 处引用但文件不存在）
  - 统一命名格式：**「通用名（别名）」**，如 `维生素B3（烟酸）.md`、`维生素B1（硫胺素）.md`、`辅酶Q10（泛醌）.md`
  - 同一系列节点保持命名风格一致（如 B 族维生素统一用"维生素Bn（别名）"，矿物质/脂类等有常用别名的节点同理）

#### 1.3 识别孤立节点

- 检查每个 wiki 页面的 `[[]]` 双链：
  - **入链为 0**：没有任何其他页面链接到它 → 孤立节点
  - **出链 < 2**：自身链接到其他页面少于 2 个 → 弱连接节点
- 将孤立和弱连接节点记入"关联补全候选"
- **孤立节点的处理方式**：搜索 wiki 中提到该节点相关关键词的页面，在合适位置添加 `[[]]` 入链，而非删除孤立节点

### 阶段二：内容充实

#### 2.1 从 source 层提取数据

对每个 🔴/🟡 节点，按以下顺序查找内容来源：

1. **匹配 source 页面**：在 `source/` 目录中搜索与节点主题相关的来源页
   - 读取 source 页面，提取与该节点相关的段落、数据、结论
   - 优先引用 source 页面中已标注的关键观点和证据
2. **匹配 raw 资料索引**：通过 source 页面的元信息，定位到 `raw/` 中的原始 PDF/文档
   - 从 raw 资料中提取更详细的数据（如具体数值、表格、研究结论）
3. **提取已有 wiki 节点的相关内容**：搜索 wiki 中与该节点相关的其他页面
   - 如果相关页面已有详细论述，考虑是否应将内容迁移或交叉引用

#### 2.2 通过网络搜索补充权威数据

当 source 层和 raw 层的信息不足时，通过网络搜索补充。**严格执行来源可信度分级**：

| 优先级 | 来源类型 | 示例 |
|--------|---------|------|
| ✅ **首选** | 权威机构官方发布 | WHO、FAO、中国营养学会、国家卫健委、FDA、EFSA 官方文件 |
| ✅ **首选** | 顶级学术期刊 | NEJM、Lancet、JAMA、BMJ、AJCN、Nutrients 等 |
| ✅ **首选** | 官方指南与共识 | 各学科临床指南、专家共识、国家标准 |
| ⚠️ **可用** | 高质量科普平台 | 丁香医生（医学科普团队审核）、科学松鼠会 |
| ⚠️ **可用** | 教材与专著 | 已出版的大学教材、学术专著（如本库 raw 中的参考书） |
| ❌ **禁止** | 未经验证的自媒体 | 微信公众号（非官方认证）、知乎个人回答、百家号等 |
| ❌ **禁止** | 商业推广内容 | 带有明显产品营销意图的文章 |
| ❌ **禁止** | 无来源的数据 | 无法追溯原始出处的任何数据或结论 |

> **核心原则**：宁可标注"待补充"也不使用未经验证的来源。每条补充数据必须标注出处。

#### 2.3 节点内容标准

充实后的节点应达到 [[知识点创建规范]] 中的标准。

### 阶段三：合并与拆分

#### 3.1 合并规则

**触发条件**：两个节点内容完全一致或高度重叠（重叠度 > 70%）

**合并策略**：

| 场景 | 操作 | 示例 |
|------|------|------|
| A 和 B 内容完全一致 | 保留更规范的文件名，删除另一个，将所有入链重定向到保留页 | `临床营养.md` 与 `临床营养治疗.md` 内容相同 → 合并为 `临床营养.md` |
| A 涵盖 B 的全部内容 | 将 B 的独特内容合并到 A，删除 B，更新所有链接 | `蛋白质.md` 涵盖 `蛋白质摄入量.md` → 合并 |
| A 和 B 侧重点不同但有重叠 | 重新界定各自范围，去重重叠部分，建立双向链接 | `减重.md` 与 `体重管理.md` → 明确边界并互链 |

**合并操作清单**（每次合并必须完成）：
- [ ] 确定保留页和删除页
- [ ] 将删除页的独特内容迁移到保留页
- [ ] 在保留页中标注内容来源（合并自哪个页面）
- [ ] 搜索所有 wiki 页面中对删除页的 `[[]]` 引用，替换为保留页
- [ ] 删除空文件
- [ ] 检查是否有其他页面引用了删除页，修复断链

#### 3.2 重命名规则

**触发条件**：文件名与内容不一致（如文件名"烟酸"但内容首行为"维生素B3"），或同目录下命名风格不统一

**重命名策略**：
- 统一为「通用名（别名）」格式，如 `维生素B3（烟酸）.md`
- 同一系列节点保持命名风格一致（如 B 族维生素用"维生素Bn（别名）"，其他有常用别名的节点同理）

**重命名操作清单**（每次重命名必须完成）：
- [ ] 读取原文件完整内容
- [ ] 创建新文件（更新标题行中的名称）
- [ ] 删除旧文件
- [ ] **全局搜索替换**：将整个 wiki 中所有 `[[旧名]]` 引用替换为 `[[新名]]`，包括 `[[旧名|显示文本]]` 格式
- [ ] 验证旧链接在 wiki 中已清零（无残留引用）

> **重命名必须全局替换链接**：Obsidian 的 `[[]]` 链接基于文件名匹配，重命名后所有旧链接会断开。必须用 Grep 全局搜索旧文件名，逐一替换为新文件名。

#### 3.3 拆分规则

**触发条件**：一个节点包含多个独立主题，每个主题都值得独立展开

**拆分策略**：

| 场景 | 操作 | 示例 |
|------|------|------|
| 一个页面混杂了 A、B、C 三个独立概念 | 拆为 `A.md`、`B.md`、`C.md`，原页面转为索引页 | 混合了"筛查+评估+诊断"的页面 → 拆为独立页面 |
| 一个页面过长（>3000字）且包含多个子主题 | 按子主题拆分为多个页面，父页面保留概述 + 链接 | 某疾病页面包含"病因+诊断+治疗+营养" → 按维度拆分 |
| 同一概念在不同目录下重复出现 | 确认是否为同一概念，若是则合并；若语境不同则保留但建立链接 | `婴幼儿营养.md` 在两个目录下 → 确认后合并或互链 |

**拆分操作清单**（每次拆分必须完成）：
- [ ] 确定拆分点和子页面命名
- [ ] 创建子页面，迁移对应内容
- [ ] 原页面保留概述 + 子页面链接
- [ ] 更新所有指向原页面的链接（按需更新到具体子页面）

#### 3.4 目录结构优化

- 合并拆分后，检查目录结构是否合理：
  - 同一层级的节点是否属于同一分类维度
  - 是否存在"应该并列但不在同一目录"的节点
  - 是否存在"目录层级过深"（超过4层）的情况
- 必要时调整文件位置，并更新所有引用路径

### 阶段四：关联补全

#### 4.1 建立交叉链接

对每个 wiki 页面执行关联检查：

1. **横向关联**：链接到同层级的相关概念（如同属"矿物质"的各元素）
2. **跨域关联**：链接到其他领域中相关的概念（如"钙"链接到"骨质疏松症"、"膳食指南"）

**链接质量标准**：
- 每个页面至少有 **3-5 个有意义的出链**
- 链接必须是有实质关联的，禁止为凑数而随意添加
- 优先链接到已存在的 wiki 页面；如需链接尚不存在的概念，先创建占位页（标记为 🔴）

#### 4.2 补全关联的优先级

按以下顺序处理：

| 优先级 | 操作 | 说明 |
|--------|------|------|
| P0 | 修复空链接 | 页面中 `[[]]` 引用了不存在的页面 → 按 [[断链检查与知识补充工作流]] 的分类策略处理（同物异名改链、知识缺口创建、无关引用清理） |
| P1 | 消除孤立节点 | 为入链为 0 的节点建立至少 2 个入链 |
| P2 | 补全弱连接 | 为出链 < 2 的节点补充有意义的关联 |
| P3 | 强化跨域链接 | 在不同领域之间建立更多有价值的交叉引用 |

#### 4.3 清理纯索引页

- 扫描 wiki 中的**纯索引页**（页面内容仅为链接列表，没有实质性原创论述、数据或分析）
- **直接删除**纯索引页，本知识库不使用索引页
- 删除后必须检查所有 wiki 页面中对被删除索引页的 `[[]]` 引用，移除断链
- 如删除后留下空目录，一并清理

## 执行策略

### 分批执行原则

本工作流涉及大量文件操作，**严禁一次性全量执行**。按以下策略分批处理：

```
第 1 批：选定一个子目录（如 wiki/营养素/矿物质/）→ 完整走通 阶段一~四
         ↓ 验证产出质量
第 2 批：扩展到相邻目录（如 wiki/营养素/ 全部）
         ↓ 确认流程稳定
第 3 批：按优先级逐步覆盖所有目录
```

### 批次优先级建议

| 优先级 | 目录 | 理由 |
|--------|------|------|
| P0 | `wiki/营养素/` | 基础概念，被大量其他节点引用，补充后收益最大 |
| P1 | `wiki/营养学基础/` | 上层概念，是其他节点的根基 |
| P1 | `wiki/疾病营养/` | 节点数量多，与营养素有大量交叉关联 |
| P2 | `wiki/膳食/` | 实践导向，与营养素和疾病均有联系 |
| P2 | `wiki/临床营养/` | 专业性强，与其他领域关联度高 |
| P3 | `wiki/前沿交叉/`、`wiki/食品安全/`、`wiki/食物/`、`wiki/人群营养/` | 节点数量相对少或独立性强 |

### 任务下达规范

- **具体指定目录**：不要说"帮我整理 wiki"，要说"请对 `wiki/营养素/矿物质/` 执行 Wiki 整理工作流"
- **可指定阶段**：如"只执行阶段一（盘点与诊断）"，先看诊断清单再决定后续
- **可指定类型**：如"只处理空节点和骨架节点，不动有内容节点"

## 关键原则

- **先诊断再动手**：每次先输出诊断清单，确认后再执行修改操作
- **最小破坏原则**：合并拆分时优先保留已有内容，不主动删除有价值的信息
- **来源可追溯**：所有新增内容必须标注来源（source 页面或权威外部来源）
- **链接有意义**：宁可少链接也不随意堆砌无意义关联
- **分批验证**：每批完成后检查产出，确认质量再继续下一批

## 常见错误

- **一次性全量整理** → 难以验证质量，出错后无法回滚，应分批执行
- **合并时丢失独特内容** → 合并前必须仔细对比两个页面的差异部分
- **拆分后忘记更新引用** → 导致大量断链，必须执行全局链接替换
- **使用未验证的网络数据** → 违反来源可信度分级规则，污染知识库
- **随意添加无意义链接** → 知识网络质量下降，应确保每个链接都有实质关联
- **跳过诊断直接修改** → 缺乏全局视角，可能导致不必要的重复操作
- **删除索引页后未修复断链** → 其他页面中对已删除索引页的 `[[]]` 引用会变成无效链接，必须逐一清理
- **frontmatter source 格式错误** → 在 source 链接后附加 `（待补充）` 等文字会破坏 YAML 解析，应使用注释或直接不写
- **充实节点时过度展开** → 篇幅远超同目录现有节点，破坏风格一致性，应参照已有节点的篇幅和风格
- **重命名文件后未全局替换链接** → 导致大量 `[[旧名]]` 断链，必须 Grep 全搜索旧名并逐一替换
- **同目录下命名风格不统一** → 如 B 族维生素中"烟酸"与"维生素B1"混用，应统一为「通用名（别名）」格式

## 产出物模板

每批整理完成后，应输出：

```markdown
## 整理报告 - [目录名]

### 盘点结果
- 扫描节点数：XX
- 空节点（🔴）：X 个 → 已充实 X 个
- 骨架节点（🟡）：X 个 → 已充实 X 个
- 有内容节点（🟢）：X 个 → 已优化 X 个

### 合并与重命名操作
| 保留页/新文件名 | 删除/原文件名 | 操作说明 |
|--------|-----------|---------|

### 拆分操作
| 原页面 | 拆分出的新页面 | 说明 |
|--------|-------------|------|

### 关联补全
- 新增交叉链接：XX 条
- 修复空链接：X 个（其中创建缺失节点 X 个）
- 重命名后全局替换链接：XX 处（涉及 XX 个文件）

### 未完成项
- [ ] 待后续处理的问题清单
```
