# Wiki 整理工作流

## 定义

对 [[Wiki层]] 已有页面进行系统性维护：**盘点诊断 → 内容充实 → 合并拆分 → 关联补全**，是知识库从"有内容"到"高质量知识网络"的关键跃迁。与 [[Ingest工作流]]（正向编译）和 [[断链检查与知识补充工作流]]（断链修复）互补。

## 节点分类

| 级别 | 标记 | 判定 | 处理 |
|------|------|------|------|
| **空节点** | 🔴 | 仅有 frontmatter 和标题，无实质内容（<30字正文） | 从 source 层提取内容充实 |
| **骨架节点** | 🟡 | 有标题结构和少量要点，缺乏段落展开和数据（<200字正文） | 同上前述充实 |
| **有内容节点** | 🟢 | 有完整段落、数据表格或专业论述 | 检查关联和交叉引用 |

同时识别：**重复/相似节点**（合并候选）、**同物异名**（需统一命名为「通用名（别名）」格式）、**孤立节点**（入链为 0 或出链 < 2，需建立关联，不删除）。

## 内容充实

从 `source/` → `raw/` → 已有 wiki 节点 → 网络（仅当不足时）逐级提取内容，创建标准参照 [[知识点创建规范]]。

## 合并与重命名

| 操作 | 触发条件 | 流程 |
|------|---------|------|
| **合并** | 两节点内容完全一致或高度重叠（>70%） | 保留更规范的文件名 → 迁移独特内容到保留页 → 全局替换所有 `[[旧名]]` 引用 → 删除空文件 |
| **重命名** | 文件名与内容不一致，或同目录命名风格不统一 | 统一为「通用名（别名）」格式 → 创建新文件 → 删除旧文件 → **全局替换链接** |
| **拆分** | 一个节点混杂多个独立主题，或 >3000字且含多个子主题 | 拆分为独立页面 → 原页面保留概述 + 链接 → 更新所有指向原页面的引用 |

> 重命名/合并/拆分后必须全局搜索验证：Grep 旧文件名，确认 wiki 中已无残留引用。

## 关联补全

| 优先级 | 操作 | 说明 |
|--------|------|------|
| P0 | 修复空链接 | 页面中 `[[]]` 引用了不存在的页面 → 按 [[断链检查与知识补充工作流]] 处理 |
| P1 | 消除孤立节点 | 为入链为 0 的节点建立至少 2 个入链 |
| P2 | 补全弱连接 | 为出链 < 2 的节点补充有意义的关联 |
| P3 | 强化跨域链接 | 不同领域之间建立有价值的交叉引用 |

> **不使用纯索引页**：本知识库不创建和保留纯索引页（仅含链接列表的页面），应直接删除并清理相关引用。

## 执行策略

**自动连续执行**，按目录优先级逐批处理：

```
第1批：选定一个子目录（如 wiki/营养素/矿物质/） → 完整走通诊断→充实→合并→关联
       ↓ 验证产出质量
第2批：扩展到相邻目录（如 wiki/营养素/ 全部）
       ↓ 确认流程稳定
第3批：按优先级逐步覆盖所有目录
```

| 指标 | 上限 |
|------|------|
| 每批处理节点数 | ≤ 30 |
| 每批合并/重命名操作 | ≤ 10 |

目录优先级：`营养素/` → `营养学基础/` → `疾病营养/` → `应用场景/` → `人群营养/` → `膳食/` → `食物分类/`

## 收尾检查与提交

- 每批整理完成后，检查本次修改文件的命名、frontmatter、双链、正文结构和全局替换结果
- 合并、重命名、拆分后的引用必须再次验证，确认没有明显残留问题
- 确认本批文件无问题后，执行一次 git 提交和 push

## 关键原则与常见错误

| 错误 | 后果 | 正确做法 |
|------|------|---------|
| 一次性全量整理 | 难以验证质量 | 分批执行 |
| 合并时丢失独特内容 | 信息损失 | 合并前仔细对比差异 |
| 拆分后忘记更新引用 | 产生大量断链 | 必须执行全局链接替换 |
| 使用未验证的网络数据 | 污染知识库 | 严格执行来源可信度分级 |
| 重命名后未全局替换链接 | `[[旧名]]` 全部断链 | Grep 全搜索旧名逐一替换 |
| frontmatter source 格式错误 | YAML 解析失败 | 不附加 `（待补充）` 等文字 |
| 充实节点时过度展开 | 破坏风格一致性 | 参照同目录已有节点篇幅 |
