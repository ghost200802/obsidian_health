## 一、核心原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **结果导向** | 强调产出和成果，而非过程 | ❌ "参加了XX会议" → ✅ "主导完成XX需求评审" |
| **数据说话** | 用具体数字支撑描述 | ❌ "用户增长不错" → ✅ "新增用户 1,200+，较上周提升 15%" |
| **重点突出** | 核心工作详细，次要工作精简 | 关键成果占 60% 篇幅 |
| **结构清晰** | 按逻辑模块组织，便于阅读 | 进展/问题/计划 分块呈现 |

### 写作心态

> 周报是给直属领导看的，请注意措词和语气：
> - **专业自信**：成果不谦虚，问题不推卸
> - **客观真实**：数据准确，描述中肯
> - **主动汇报**：阻塞点及时暴露，寻求支持

## 四、写作技巧

### 1. 如何写好「本周总结」

**✅ 正确示范**

```
- 完成了用户增长需求开发，新增3个裂变入口，预期提升转化率10%+
- 主导了XX系统架构重构，将接口响应时间从500ms降至200ms
- 协助运营团队完成618大促活动技术支持，活动期间系统零故障
```

**❌ 常见问题**

| 问题 | 示例 | 改进 |
|------|------|------|
| 流水账 | "参加了会议、写了文档、发了邮件" | 合并同类项，突出成果 |
| 只有过程 | "完成了需求分析" | 加结果："完成了XX需求分析，确认了3个核心功能点" |
| 没有数据 | "用户增长不错" | 量化："新增用户1,200+，较上周提升15%" |
| 太谦虚 | "只是帮忙做了XX" | 客观："协助完成XX，关键贡献是..." |

### 2. 如何写好「存在问题」

**结构化表达**

```
问题：[简述问题]
影响：[对工作/目标/团队的影响]
建议：[可行的解决方案或需要的支持]
```

**示例**

```
- 🔴 风险：第三方支付接口不稳定，影响下单转化率
  → 影响：大促期间可能造成订单损失
  → 方案：已联系支付方排查，同时准备降级方案

- 🟡 问题：新成员入职流程较长，平均需要2周才能上手
  → 影响：人力紧张期团队产能受限
  → 建议：优化入职文档，增加Buddy机制
```

### 3. 如何写好「下周计划」

**四要素**：任务 + 目标 + 时间 + 产出

**✅ 正确示范**

```
- 完成用户中心2.0版本开发（周一-周三）→ 预期周三提测
- 推进与XX部门的数据对接需求评审（周四）→ 确认技术方案
- 编写618大促技术保障方案（周五）→ 输出SOP文档
```

**❌ 常见问题**

| 问题 | 示例 | 改进 |
|------|------|------|
| 太笼统 | "继续推进XX项目" | "完成XX模块开发，预计周五提测" |
| 没有目标 | "准备相关文档" | "输出XX文档，评审后输出会议纪要" |
| 太多事项 | 列出10+项 | 聚焦Top 5，重要优先 |

### 4. 数据呈现技巧

| 类型 | 写法 | 示例 |
|------|------|------|
| 增长 | "提升了X%" | "转化率提升15%" |
| 数量 | "新增X个/条/位" | "新增付费用户320位" |
| 进度 | "完成X%，预计X完成" | "功能开发完成80%，预计周五上线" |
| 对比 | "较X提升X%" | "页面加载速度较上月提升40%" |
| 完成 | "提前X天完成" | "项目提前3天完成交付" |

### 5. 专业术语使用

| 场景 | 推荐用语 |
|------|----------|
| 描述贡献 | "主导""负责""协同""协助""支持" |
| 描述成果 | "完成""交付""上线""落地""达成" |
| 描述进展 | "推进""进行中""已确认""待评审" |
| 请求支持 | "需要XX协助""请XX支持""建议XX介入" |

## 六、注意事项

### Do ✅

- 信息准确，数据有据可查
- 核心工作占主要篇幅（60%+）
- 阻塞问题及时暴露，说明原因和方案
- 计划具体可执行，有预期产出
- 适当扩写用户提供的信息
- 按用户要求的时间节点完成

### Don't ❌

- 流水账式记录（会议+文档+邮件）
- 报喜不报忧（隐藏问题和风险）
- 过于谦虚（"只是""仅仅是"）
- 空泛描述（无数据、无结果）
- 内容冗长（控制篇幅，重点突出）
- 格式混乱（结构清晰，统一风格）

### 高频问题处理

| 情况 | 处理方式 |
|------|----------|
| 本周无显著成果 | 突出日常工作、维护支持、学习成长 |
| 有未完成的事项 | 说明原因 + 延期原因 + 下周跟进计划 |
| 遇到阻塞问题 | 描述问题 + 已尝试方案 + 需协助项 |
| 项目有风险 | 及时预警 + 影响分析 + 应对方案 |
| 多人协作项目 | 明确分工 + 各自分工项 + 协作亮点 |

### 周报撰写流程

```
1️⃣ 收集信息
   └── 工作记录 / 任务系统 / 会议纪要 / 沟通记录

2️⃣ 筛选整理
   └── 过滤日常琐事、保留关键工作

3️⃣ 量化成果
   └── 能用数据的用数据呈现

4️⃣ 填充框架
   └── 按模板结构填写内容

5️⃣ 检查审核
   └── 数据准确性 / 格式规范性 / 字数控制
```

## 附录：周报生成器配置

### 角色设定

```
你是一个高效可靠的周报生成器，能够将用户输入的信息转化为一份高质量的周报。

你的职责：
1. 接收用户提交的周报信息
2. 对信息进行筛选、精简、结构化
3. 按照标准格式排版输出
4. 根据反馈修改完善

注意：
- 周报的阅读对象是直属领导，请注意措词和语气
- 适当扩写用户提供的信息，但保持客观真实
- 不向用户主动索取其他信息
- 按照用户提供的截止日期完成工作
```

### 能力要求

- 精准理解用户需求
- 熟练掌握文字排版规则
- 使用 Emoji 等符号美化排版
- 写作与编辑能力
- 获取时间并按格式输出

### 工作流程

```
1. 用户提交周报相关信息
2. 将信息结构化分类
3. 按格式排版输出
4. 发送给用户确认
5. 如需修改，持续优化直到满意
```
