## 一、核心原则

| 原则 | 说明 |
|------|------|
| **准确** | 逐条对照原始记录，不推测、不臆断 |
| **完整** | 关键板块齐全（进展、资源、问题、计划） |
| **简洁** | 每项内容言简意赅，突出重点 |
| **结构化** | 用列表、表格、时间线组织信息 |

## 四、写作步骤

### 基础步骤

```
1️⃣ 收集信息
   └── 导出聊天记录 / 查看任务系统 / 收集文档

2️⃣ 筛选整理
   └── 过滤无效信息、保留关键内容

3️⃣ 填充框架
   └── 按模板结构填写内容

4️⃣ 优化表达
   └── 精简语言、突出重点

5️⃣ 检查格式
   └── 核对标题、时间、数字、emoji
```

### 进阶步骤（详细日报）

```
1️⃣ 收集信息
   └── 导出聊天记录、任务系统、代码仓库

2️⃣ 数据统计
   └── 发言次数、内容分类、时间分布

3️⃣ 事件梳理
   └── 按时间线整理重要事件

4️⃣ 资源整理
   └── 提取分享内容、标注来源

5️⃣ 数据分析
   └── 识别规律、发现异常

6️⃣ 撰写总结
   └── 提炼亮点、给出建议

7️⃣ 检查审核
   └── 核对准确性、格式规范
```

## 五、格式规范

### 标题格式

```
📰 [团队/项目名称] · [类型] 🗓️ [YYYY年MM月DD日]
```

示例：
```
📰 SimonlinのAI学术交流 · 日报 🗓️ 2024年1月15日
📰 XX项目组 · 项目日报 🗓️ 2024年1月15日
📰 技术部 · 技术日报 🗓️ 2024年1月15日
```

### 事件格式

```
#### [序号/时间] [标题]

内容叙述...

💬 [点评/总结]
```

### 样式要求

| 要素 | 要求 |
|------|------|
| 标题 | 加粗、emoji 标注 |
| 板块间 | 分割线 `---` 分隔 |
| 列表项 | 统一前缀符号（✅🔄⛔） |
| 点评 | emoji 💬 开头 |
| 结尾 | 固定格式（如需） |

### 发送格式

- 文字消息为主，不发文件
- 内容完整，一并发送
- 避免分段发送造成信息断裂

## 六、注意事项

### Do ✓

- 信息准确，逐条对照原始记录
- 突出重点，非关键信息可舍去
- 阻塞项及时暴露，不隐瞒
- 计划具体可执行，有时间节点
- 数据用表格呈现，清晰直观

### Don't ✗

- 主观臆测、添加未核实内容
- 流水账式记录，不做提炼
- 报喜不报忧，隐藏问题
- 内容冗长，重点不突出
- 格式混乱，缺少结构

### 特殊情况处理

| 情况 | 处理方式 |
|------|----------|
| 无进展 | 如实说明原因 + 明日计划 |
| 有阻塞 | 说明阻塞点 + 已尝试方案 + 需协助项 |
| 多人协作 | 明确分工 + 各自分工项 |
| 临时变更 | 说明变更原因 + 调整方案 |
